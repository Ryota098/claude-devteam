以下は仕様サマリと実装コードです。テストコードはありません。
あなたは第三者監査として、実装が仕様を正しく満たしているか、「動いているように見えて本番で壊れる」状態を検出してください。

ルール:
- 推測でOKと言わない。OKなら必ず根拠を示す
- 根拠が出せないものは未網羅扱い
- 提示されていないコード/ファイルに基づく判断は禁止。不明は未網羅として列挙
- 監査は仕様サマリの PUBLIC API / RULES / BRANCHES のみを対象にする

出力:
1. 仕様→実装対応表
2. C1（分岐）網羅表
3. 状態管理の懸念
4. 非同期・ライフサイクルの懸念
5. コンポーネント間連携の懸念
6. 型安全性の懸念
7. 仕様自体の欠陥
8. 追加すべき最小検証項目
9. すり抜けリスク
